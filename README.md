## 主なデータ処理と課題
- 温度の季節要因をFFTを使ったバンドパスフィルタで除去
- 日照時間の季節要因は月日ごとの日照時間の全期間での最大値から理論値を近似して除去
- 上記処理を行った温度に対して、前項日照時間、および降雨量、平均風速との各々相関係数を求めて依存成分を除去
- 課題1. いろいろ試したが、26年分のデータから最大日照時間を得ることができず（右上のグラフ、赤いグラフが分布の上渕をなぞる必要がある）、それにはより複雑な処理が必要
- 課題2. 日照時間と温度の相関について日照当たりのエネルギーにも季節要因があり、季節を通じて同じ係数ではまずいことがわかった  

### :sparkles:バイブコーディングによるアップデート
- 課題１の対策として、最大日照時間を緯度から求める理論値にすること。課題2については月ごとに回帰係数を求めることで対応
- これらの方針を、AIに対して、旧コードをそのまま示すのではなく、上記内容を反映させた旧コードの概要を言葉で表して指示し、AIからの質問や提案に対しても原則言葉のみですり合わせをしてからコードを吐き出させた<br>
<b>例（最初の指示）</b>(使用AI Clude.ai Sonnet 4.6)
```
気温の長期トレンドを見るため、25年分の日毎の平均気温データに対して、季節要因と気象要因の補正を行うプログラムを作成してください。
季節要因は温度の月日ごとの平均から求めようとしていますが、気象要因のうち日照時間にも季節要因があるので、クロストークがおきると思います。これらをどう処理するか悩んでいます。
日照時間が理論値が求まることがわかったので、日ごとの日照時間の理論値に対する割合を求め、月日ごとの平均値に依存する温度は温度の季節要因に含まれると仮定して、偏差分を年ごとの温度補正に使う案を考えていますが、どうでしょうか？
```
